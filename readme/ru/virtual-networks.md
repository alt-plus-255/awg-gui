# Виртуальные сети (VN)

**Языки:** [Русский](virtual-networks.md) | [English](../en/virtual-networks.md) | [README](../../README.md)

Конфиги типа **Виртуальная сеть** (`virtual_network`) создают изолированную LAN между VPN-клиентами без выхода в интернет через сервер.

## Политики доступа

При создании VN задаётся политика **`vn_policy`**:

| Политика | Поведение |
|----------|-----------|
| **Все видят всех** | Любой пир в конфиге может обращаться к любому другому |
| **Изоляция** | Пиры по умолчанию изолированы; доступ настраивается правилами зон |

## Зоны доступа

В режиме изоляции панель позволяет создавать **зоны** и правила «кто видит кого». Пиры в одной зоне (или связанные правилами) могут обмениваться трафиком; остальные — нет.

## Исключения между пирами

Можно задать **исключения** (exclusions) — точечные разрешения или запреты между конкретными парами пиров поверх общей политики.

## Граф связей (Connections)

Страница **Connections** (или соответствующий раздел панели) показывает граф связей между пирами VN: кто с кем может обмениваться трафиком согласно политике, зонам и исключениям.

## Клиентские конфиги

Для VN генерируются готовые клиентские `.conf` / QR с подсетью виртуальной сети. Резолвер split-tunnel для VN **не используется** — см. [resolver.md](resolver.md).

Создание конфигов и привязка пиров — на странице [Конфиги и пиры](configs-and-peers.md).

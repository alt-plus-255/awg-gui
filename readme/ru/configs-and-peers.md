# Конфиги и пиры

**Языки:** [Русский](configs-and-peers.md) | [English](../en/configs-and-peers.md) | [README](../../README.md)

Страница **Конфиги и пиры** — основной интерфейс управления AmneziaWG: интерфейсы, ключи, AllowedIPs и привязка клиентов.

## Несколько конфигов AWG

Можно создать до **20** конфигов AmneziaWG. Каждый конфиг получает:

- свой интерфейс (`awg0`, `awg1`, …);
- отдельную внутреннюю подсеть;
- UDP-порт из диапазона **51820–51839**.

Типы конфигов:

| Тип | Назначение |
|-----|------------|
| **Сервер** | Классический VPN: клиенты выходят в интернет через сервер (или split-tunnel через [резолвер](resolver.md)) |
| **Виртуальная сеть** | Изолированная LAN между пирами — см. [virtual-networks.md](virtual-networks.md) |

При создании конфига панель автоматически выделяет свободный `iface` и UDP-порт.

## Пиры (клиенты)

**Пир** (`vpn_client`) — отдельная сущность с ключами и именем. Пир **не привязан** к конфигу навсегда:

- **Привязка** — пир добавляется в выбранный конфиг, генерируется конфигурация для клиента;
- **Отвязка** — пир удаляется из конфига, но запись пира остаётся в панели;
- **Перепривязка** — пир можно привязать к другому конфигу (другая подсеть, другой тип).

Непривязанные пиры отображаются отдельно — их можно позже привязать к любому конфигу.

## Экспорт конфигурации

Для каждого привязанного пира доступны:

- скачивание файла **`.conf`** (AmneziaWG / WireGuard);
- **QR-код** для импорта в мобильное приложение.

После смены endpoint, UDP-порта или настроек резолвера переэкспортируйте или заново импортируйте конфиг на устройствах.

## Настройки панели

В **Settings** можно изменить endpoint панели, порт и [webhook сбоев](webhook.md).

Параметры VPN endpoint (публичный IP/DNS, UDP-порт AWG) также доступны через [CLI](cli.md#публичный-vpn-endpoint).

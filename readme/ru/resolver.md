# Резолвер

**Языки:** [Русский](resolver.md) | [English](../en/resolver.md) | [README](../../README.md)

Страница **Резолвер** в панели AWG-GUI настраивает маршрутизацию для конфигов типа **Сервер**. Для виртуальных сетей резолвер **не используется**.

## Общая схема

1. На VDS в контейнере AWG работает **sing-box** с FakeIP и ruleset-списками.
2. Community-списки ([allow-domains](https://github.com/itdoginfo/allow-domains)) скачиваются на диск (`rulesets/*.srs`) — страница **Настройки списков**.
3. Для каждого серверного конфига выбирается **Подключение** — точка выхода в интернет (VLESS, VMess, подписка и т.д.).
4. В клиентском `.conf` при включённом резолвере: `DNS = gateway` и `AllowedIPs` по выбранному режиму.

## Два режима маршрутизации

### 1. Полный туннель на VDS (`vds_split`) — по умолчанию

**В панели:** «Полный туннель на VDS (AllowedIPs 0.0.0.0/0)».

**Клиентский `.conf`:**
```
DNS = gateway
AllowedIPs = 0.0.0.0/0, ::/0
```

**Поведение:**

| Трафик | Маршрут |
|--------|---------|
| Весь трафик клиента | Через AmneziaWG на VDS |
| Домены из списков (FakeIP) | sing-box → выбранное **Подключение** |
| Сайты вне списков, Speedtest, 2ip.ru | С **IP сервера VDS** |
| IP-CIDR из community-списков | Проксируются на VDS (полная поддержка) |

**Когда выбирать:** нужен полноценный VPN через VDS, но заблокированные ресурсы (Telegram, YouTube…) выходят через отдельное upstream-подключение, а не с IP VDS.

### 2. Split-tunnel (`client_split`) — тестовый режим

> **Split-tunnel сейчас в тестовом режиме.** Режим работает и доступен в панели, но поведение и ограничения могут дорабатываться. Для стабильной эксплуатации рекомендуется **полный туннель на VDS**.

**В панели:** «Split-tunnel (только списки через VDS)».

**Клиентский `.conf`:**
```
DNS = gateway
AllowedIPs = 198.18.0.0/15, <gateway>/32
```
Плюс свои подсети из поля «Свои подсети», если добавлены.

**Поведение:**

| Трафик | Маршрут |
|--------|---------|
| Домены из списков (FakeIP) + DNS | Через туннель → **Подключение** |
| Весь остальной трафик | **Напрямую с клиента** (SIM/Wi‑Fi) |
| 2ip.ru, Speedtest | IP **клиента**, не VDS |
| IP-CIDR из community-списков | **Не** проксируются (прямой IP без DNS → FakeIP не срабатывает) |

**Когда выбирать:** через VPN нужны только ресурсы из списков; остальной интернет — без туннеля, с IP оператора/Wi‑Fi.

**Ограничение split:** для полной поддержки IP-CIDR из community-списков используйте полный туннель на VDS.

## Списки и подключения

- **Community-списки** — YouTube, Meta, Telegram, Discord и др.; синхронизация в **Настройки списков** (интервал по умолчанию 6 ч).
- **Свои домены и подсети** — на карточке конфига на странице Резолвер.
- **Подключения** — отдельная страница; без выбранного подключения резолвер не применяется.
- **Блокировать QUIC** — форсирует TCP для FakeIP-доменов (аналог reject QUIC для YouTube, но для всех выбранных списков).

## AmneziaWG: переимпорт конфига

**iPhone и Android:** после **включения**, **выключения** или **смены режима** резолвера:

1. Удалите сервер в AmneziaWG.
2. Заново импортируйте QR или `.conf` из панели.

Без переимпорта в конфиге останутся старые `AllowedIPs` — списки (Telegram, YouTube, Meta…) не заработают.

**Проверка на телефоне:**

| Режим | 2ip.ru | Домен из списка |
|-------|--------|-----------------|
| Полный туннель | IP VDS | Через VPN-подключение |
| Split-tunnel | IP клиента | Через VPN-подключение |

## Диагностика и типичные проблемы

- **iPhone:** отключите iCloud Private Relay на время проверки.
- **Android:** отключите Private DNS / DoH; при сбоях Telegram — очистите кэш приложения.
- Не проверяйте «весь VPN» через Speedtest — откройте сайт из списков.
- Кнопка **Диагностика** проверяет sing-box, наличие `.srs` на диске и DNS → FakeIP для каждого включённого списка.

## sing-box в образе AWG

Резолвер использует [sing-box](https://github.com/SagerNet/sing-box) внутри контейнера AWG. В production-сборке sing-box уже включён в образ; при dev-сборке tarball скачивается установщиком — см. [install.md](install.md#sing-box-vendor-только-dev-сборка).

Подробности лицензии и брендинга sing-box — в [README](../../README.md#sing-box-и-брендинг) и [NOTICE.md](../../NOTICE.md).
